# 文件创建与放置
首先创建一个格式为 **年-月-日-标题.后缀** 的文件并将其放在根目录的 **_posts** \
注意:**后缀**必须是**md**\
如果您想节省创建文件的时间,可以使用插件: [Jekyll-Compose](https://github.com/jekyll/jekyll-compose)

# 前言 | Front Matter
您可以在文件的顶部填写如下 **Front Matter**

```yaml
---
title:              # ✅【必填】文章标题，会显示在网页标题和文章头部
description:        # ✅【推荐】页面的描述，用于 SEO 摘要（搜索引擎抓取摘要用）
author:             # ❌【可选】文章作者，不写则默认用 _config.yml 的 social.name
date:               # ✅【必填】发布日期，如 2025-07-17，会用来排序文章时间线
categories:         # ❌【推荐】文章所属分类，支持单个或列表，自动生成目录页
tags:               # ❌【推荐】标签，用于文章检索和关联阅读，支持多个
toc:                # ❌【推荐】是否显示目录（Table of Contents），true / false，默认 true
pin:                # ❌【可选】是否置顶文章，true 表示置顶（Chirpy 支持置顶）
math:               # ❌【可选】是否启用 KaTeX 数学公式支持，true 启用
mermaid:            # ❌【可选】是否启用 Mermaid 流程图支持，true 启用
comments:           # ❌【推荐】是否启用评论功能，true/false，默认跟随 _config.yml
render_with_liquid: # ❌【进阶】是否使用 Liquid 模板引擎渲染文章，默认 true
media_subpath:      # ❌【进阶】该文章的资源（图片等）所在子路径，如 `2025/example`
image:              # ❌【推荐】用于社交平台预览图，支持下列子字段
  path:             #    图片路径，支持相对路径或外链
  lqip:             #    用于懒加载的低质量预览图路径（Low Quality Image Placeholder）
  alt:              #    图片的替代文本，用于 SEO 和无障碍阅读
---
```
